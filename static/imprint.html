Data protection

Logging of retrieval/download requests

Every time a user accesses the web site and every time a file is requested, data connected to these processes are stored in a log, including the IP-address of the originating request.

Logging of authentication data for uploading and/or modifying content

When users want to register content on the web site, they have to authenticate themselves via existing third party user profiles, currently this is possible via Google, Github or LinkedIn. By relying on such external profiles, we do not have to store detailed personal information such as person names and email addresses ourselves. We have no influence on the processes involved on side of the maintainers of these services but chose popular services which we verified to contain GDPR compliancy statements:

Google data protection statement
Github privacy statement
LinkedIn GDPR statement

The result of the authentication process via one of these Social Media profiles, however, is logged locally including the user id of that third party profile, and whenever a user stores information via the web site, his or her user id is stored alongside the content in order to identify the stored content with that user. Besides that, the sole responsibility for uploaded and registered content lies with the authenticated users uploading and registering that content.

The log is not accessible to the public and the data that is saved will be used for statistical purposes/debugging only. This information will not be disclosed to third parties.
